    函数重载（overload）¶

    函数重载简介¶

    如下函数所示：

    int add(int x, int y) {
  return x + y;
}

    这个简单的函数将两个整数相加并返回一个整数的结果。但是如果我们要求对两个浮点数相加并返回浮点数的结果，这个函数就不能实现，因为任何浮点形参都会被转换为整数，导致浮点形参丢失它们的小数值。

    如果我们要实现这两种类型的数值相加，可以定义多个名称略有不同的函数：

    int addInteger(int x, int y) {
    return x + y;
}

double addDouble(double x, double y) {
    return x + y;
}

    但是，以这种方式定义函数会使得代码变得难以维护，因为我们不仅需要定义一致的命名标准，同时还需要记住这些函数的名称。如果我们想要添加一个三个整数的函数相加，此时的函数命名就会变得更加复杂。

    C++ 使用函数重载机制来解决这个问题，函数重载允许我们在同一个作用域中定义多个相同名称的函数，但是每个函数的形参列表必须不同。因此，上述所说的添加两个不同类型的函数可以定义成如下格式：

    int add(int x, int y) { // integer version
  return x + y;
}

double add(double x, double y) { // floating point version
  return x + y;
}

int main() {

  return 0;
}

    只要编译器能够区分每个重载函数，函数就可以被重载。如果不能区分重载函数，将导致编译错误。

    函数重载的条件

    函数产生重载的条件有以下几个:

    • 需要重载的函数必须在同一个作用域中（一旦编译器在当前作用域中所需名称的函数，就会忽略掉外层作用域中的同名实体[具体代码细节描述阅读 Cpp Primer 210 页] ）
    • 函数的参数列表不同：参数个数、参数类型（不包括 typedef、类型别名、const 限定符）、参数类型的顺序
    • 函数参数是某种类型的引用和指针时，可以通过区分其指向的是常量对象还是非常量对象实现重载
    • 省略号参数可以被任务是一种独特的参数类型，它也可以实现重载
    • 普通成员函数与 const 成员函数可以重载

    函数的返回类型不用于区分重载函数，默认参数会与重载产生冲突，使用时需谨慎。

    const_cast 在重载函数的情景中最有用，这也是在实际开发中很常用的一种方式，如下所示：

    const string &shorterString(const string &s1, const string &s2) {
  return s1.size() <= s2.size() ? s1 : s2;
}

string &shorterString(string &s1, string &s2) {
  auto &str = shorterString(const_cast<const string &>(s1),
                            const_cast<const string &>(s2));
  return const_cast<string &>(str);
}

    函数匹配¶

    delete 函数¶

    看下面这个示例：

    #include <iostream>

void printInt(int x) {
  std::cout << x << '\n';
}

int main() {
  printInt(5);    // okay: prints 5
  printInt('a');  // prints 97 -- does this make sense?
  printInt(true); // print 1 -- does this make sense?

  return 0;
}

    输出结果是：

    5
97
1

    printInt(5) 显然是正确的，但是 printInt('a') 和 printInt(true) 的结果显然跟我们预想的不一样，因为在调用 printInt('a') 时，编译器会将 a 的值转换成整型值 97，将 true 转换成整型值 1，以便将函数与函数定义匹配，这些都是编译器会做的事。

    删除所有不匹配的重载¶

    删除一堆单独的重载函数是一种很好的行为，。有时候，我们只使用其类型与函数参数完全匹配的参数调用某个函数，可以使用函数模板，如下：

    #include <iostream>

// This function will take precedence for arguments of type int
void printInt(int x) {
  std::cout << x << '\n';
}

// This function template will take precedence for arguments of other types
// Since this function template is deleted, calls to it will halt compilation
template <typename T>
void printInt(T x) = delete;

int main() {
  printInt(97);   // okay
  printInt('a');  // compile error
  printInt(true); // compile error

  return 0;
}
